情绪容器 · Emotion Vessel

有些情绪不需要被拆解成结论，
只需要一个地方，安静地存在。

我们习惯让算法理解情绪、判断情绪、回应情绪。
但在这里，AI 不试图解释你。它只协助你，为一段情绪选择一个容器——一个透明的、狭小的、或被时间包裹的空间。

这个项目在做什么？
  • 把情绪当作“可被安放的经验”，而不是需要被修复的问题。
  • 用三种容器外观，表达三种不同的安放方式（透明 / 短暂 / 收纳）。
  • 将作品导入 3D 展厅，形成可浏览的“情绪陈列”。
说明：制作流程中涉及的工具/平台仅作为方法，不作为品牌展示；页面以叙事与空间隐喻为主。

入口 · 向导

你可以把它理解为“作品生成与整理流程”的说明页。

STEP 1
准备一张情绪图像

选择一张你愿意交给它的图像：可以是照片、插画、符号或抽象纹理。
它不需要“好看”，只需要“真实地属于那段情绪”。

单张图像 可抽象 / 可具象 不做诊断
STEP 2
生成 3D 高斯场景

使用任意可用的生成工具，将单图转为 3D Gaussian Splats（导出为 .ply）。
这里强调的是“把经验变成可进入的空间”，而非追求写实还原。

输出：.ply 空间化表达 可重复调整
STEP 3
精修与裁剪

你可以用一个 3D Gaussian Splats 编辑器做裁剪、对齐、去噪与构图微调。
建议把“容器气质”放在第一位：让空间看起来像一种安放方式。

裁剪 / 去噪 构图修正 导出仍为 .ply

制作 · 作品包与展厅

让“制作过程”在页面里更像作品的一部分，而不是一句带过。

为什么“导出作品包”看起来像一个 JS 文件？

因为展厅采用的是浏览器端渲染：页面需要一个脚本（JS）去加载 3D 库、读取 .ply 文件并渲染画面。
作品本体是 .ply，而 JS 负责把作品“呈现出来”（就像播放器与视频文件的关系）。

你提交作业时可以这样表述：
「作品由 3D Gaussian Splats（.ply）构成，网页端脚本负责加载与交互，形成可进入的展厅体验。」

Supersplat（或同类编辑器）链接怎么体现？

如果你需要在页面里体现“我确实用过编辑器”，最稳妥的方法不是写品牌名，而是写成： “使用 3D Splats 编辑器进行裁剪与精修”，并附上一个“工具参考链接”。

建议：工具参考链接写在 README 页面避免品牌堆叠

选择情绪容器

三个容器不对应具体作品；它们只在外观与叙事气质上不同。

容器不是风格选择，而是一种安放方式。 你不需要解释你的情绪，只需要决定它更像哪一种“被放置”的形态。

玻璃鱼缸
Glass Tank

透明，却并非敞开。
情绪被看见、被承认，但仍被温和地隔离在外。

火柴盒
Matchbox

很小，很近。
只容纳一个瞬间的感受，不延展，也不回看。

木质抽屉
Wooden Drawer

被收纳、被叠放。
情绪没有被处理，只是暂时安置，等待以后再打开。

当前选择：火柴盒 · Matchbox
很小，很近。只容纳一个瞬间的感受，不延展，也不回看。
